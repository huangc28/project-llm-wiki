<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user cwd -> Git root resolver | Untrusted current directory determines which repository receives `.llm-wiki/`. |
| filesystem -> init writer | Existing files and directories can conflict with required wiki paths. |
| CLI args -> subprocess | User arguments must not become shell strings. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `apply_init_plan` | mitigate | Preflight all required file and directory types before writes; return nonzero and print `Conflicts:` if any mismatch exists. |
| T-02-02 | Spoofing | `resolve_git_root` | mitigate | Use `git rev-parse --show-toplevel` with `shell=False`; write only under the returned root. |
| T-02-03 | Tampering | `create_file_if_missing` | mitigate | Check `path.exists()` before every `write_text`; existing files are reported under `Skipped existing paths:`. |
| T-02-04 | Elevation of privilege | Git subprocess calls | mitigate | Use argument-list subprocess calls only; do not invoke shell parsing and do not add `--target`. |
| T-02-05 | Denial of service | parent workspace candidate scan | accept | Candidate discovery is shallow immediate-child scanning only, so it cannot recurse through large workspaces. |
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| package templates -> target repo wiki | Static package assets become git-tracked files in another repository. |
| target repo README/AGENTS -> repo overview | Human-authored repo files influence generated wiki text. |
| raw policy text -> future user behavior | Policy must clearly forbid unsafe material before lint enforcement exists. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Information disclosure | raw policy templates | mitigate | Include the exact denied list forbidding secrets, credentials, auth tokens, private customer data, full logs, database exports, and generated dumps. |
| T-02-07 | Tampering | template writer | mitigate | Reuse Plan 02-01 missing-only creation; template loading only supplies content for absent files. |
| T-02-08 | Information disclosure | `build_repo_overview` | mitigate | Read only `README.md` and `AGENTS.md`; do not read manifests, logs, exports, or arbitrary files. |
| T-02-09 | Repudiation | seeded overview provenance | mitigate | Write `Sources found:` and `Sources skipped:` in `repo-overview.md` so seed origin is auditable. |
| T-02-10 | Information disclosure | secret scanning expectations | accept | D-24 defers deterministic scanning to Phase 3; Phase 2 templates must not claim scanning enforcement exists. |
</threat_model>

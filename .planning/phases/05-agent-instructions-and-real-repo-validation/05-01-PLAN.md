<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI arguments -> init planner | `--dry-run` and `--no-patch-agents` determine whether writes are allowed and whether root `AGENTS.md` is in scope. |
| Git root -> filesystem writes | All generated paths must stay under the resolved git root; root `AGENTS.md` is the only non-`.llm-wiki/` write target. |
| Existing root `AGENTS.md` bytes -> marker parser | User-owned instructions may contain invalid UTF-8, unmatched markers, duplicate markers, NotebookLM, GSD, workflow sections, or repo-specific text. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | `build_agents_patch_plan` | mitigate | Target only `git_root / "AGENTS.md"` and reject symlinks, directories, and paths resolving outside the git root. |
| T-05-02 | Tampering | marker replacement | mitigate | Replace only the inclusive `<!-- PROJECT-LLM-WIKI:START -->` to `<!-- PROJECT-LLM-WIKI:END -->` span when exactly one valid pair exists; tests compare marker-external bytes. |
| T-05-03 | Denial of Service | marker parser | mitigate | Treat invalid UTF-8, unmatched start marker, unmatched end marker, and duplicate marker pairs as conflicts with return code 2 and remediation output. |
| T-05-04 | Tampering | `run_init --dry-run` | mitigate | Dry-run prints `Managed AGENTS.md section:` and planned root `AGENTS.md` action without writing `.llm-wiki/` or `AGENTS.md`. |
| T-05-05 | Information Disclosure | root managed section content | mitigate | Insert only the short protocol text specified in D-06 through D-11; do not copy wiki-local raw policy details or project secrets. |
</threat_model>

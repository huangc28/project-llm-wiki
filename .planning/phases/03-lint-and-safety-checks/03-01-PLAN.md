<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user cwd -> Git root resolver | The current directory determines which repository's `.llm-wiki/` is inspected. |
| `.llm-wiki/` files -> lint parser | Markdown content may contain malformed links, invalid UTF-8, or path-like strings. |
| `.llm-wiki/raw/` filesystem -> raw-size check | Raw files may be large dumps or symlink-like structures that must not escape the repo. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `check_broken_wikilinks` | mitigate | Normalize only Obsidian `[[...]]` destinations and verify resolved targets under `.llm-wiki/`; broken links produce `error`. |
| T-03-02 | Information Disclosure | `collect_wiki_files` / `collect_markdown_files` | mitigate | Inspect only regular files under the resolved Git root and `.llm-wiki/`; reuse `path_is_under`, reject `../` wikilink escapes, and add symlink/outside-root fixtures proving inventory collection does not follow escapes. |
| T-03-03 | Denial of Service | `check_raw_file_sizes` | mitigate | Use byte-size metadata for raw size warnings; do not read large raw files just to determine size. |
| T-03-04 | Tampering | `run_lint` exit code | mitigate | Return 1 only for `error` findings; warning-only structural debt remains visible but non-blocking. |
| T-03-05 | Repudiation | finding fields | mitigate | Emit fixed fields `severity`, `code`, `path`, `message`, and `remediation` for every finding. |
</threat_model>

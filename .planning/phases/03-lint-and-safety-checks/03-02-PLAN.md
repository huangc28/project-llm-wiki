<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| wiki text -> secret heuristics | Markdown content may include real secrets or policy prose that mentions unsafe-material categories. |
| wiki frontmatter -> date parser | User-authored frontmatter may contain missing, malformed, or stale `updated:` values. |
| wiki code spans/blocks -> repo path checker | Wiki references to repo paths are untrusted and may point outside the current repo. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Information Disclosure | `check_secret_like_content` | mitigate | Match only high-confidence private key block and credential URL shapes; warn users to redact unsafe material. |
| T-03-07 | Denial of Service | secret scanner | mitigate | Avoid broad keyword matching that would warn on default raw policy prose and create noisy warning debt. |
| T-03-08 | Tampering | `check_stale_pages` | mitigate | Warn on stale `updated:` frontmatter but never edit dates automatically. |
| T-03-09 | Tampering | `check_repo_path_drift` | mitigate | Limit drift warnings to missing repo paths from code spans and fenced blocks; current repo files remain authoritative. |
| T-03-10 | Information Disclosure | path drift candidate handling | mitigate | Check only relative path candidates under the resolved git root; ignore URLs and `.llm-wiki/` internal paths. |
</threat_model>
